# 디자인 QA 도구 (개인용)

Figma 디자인과 로컬 웹 구현이 **얼마나 일치하는지** Claude Code가 비교해, 한 장짜리 리포트(`reports/qa-report.html`)를 만들어 브라우저로 띄워준다.

## 한 번만 — 셋업
```
npm run setup        # Playwright + Chromium 설치 (약 170MB, 최초 1회)
```
> "QA 해줘"라고 했는데 미설치 상태면 Claude가 알아서 셋업을 제안한다.

## 쓰는 법 (Claude Code에서 그냥 말하면 됨)

**단일 화면**
> 이 피그마랑 http://localhost:3000 디자인 QA 해줘
> https://figma.com/design/XXXX?node-id=1-23

**반응형 (여러 브레이크포인트)**
> 모바일/데스크톱 반응형 QA 해줘
> - 모바일(375): <Figma 모바일 프레임 URL>
> - 데스크톱(1440): <Figma 데스크톱 프레임 URL>
> 웹은 http://localhost:3000

준비물: ① Figma **프레임** URL(`node-id` 포함) ② 실행 중인 로컬 웹 URL.
정확한 비교를 위해 캡처 폭은 Figma 프레임 폭에 맞춘다(보통 자동, 모를 때만 물어봄).

## 결과
- `reports/qa-report.html` 가 자동으로 열린다.
- 리포트 구성: **중립 헤더(차이 요약)** → **시각 비교(스와이프/오버레이/나란히 — 리포트의 주인공)** → **차이 목록(로케이터 포함)** → **확인 필요 / 거의 일치**.
- 차이 정도: 🟠 주목(우선 검토) · ▫️ 미세(넘어가도 무방할 수 있음) · ◇ 확인 필요.
- **합격/불합격을 판정하지 않는다.** 픽셀 100% 일치는 어렵기 때문 — 도구는 차이를 크기순으로 보여줄 뿐, **고칠지 말지는 디자이너가 판단**한다.

## 알아두면 좋은 것
- **스타일/레이아웃 QA**에 집중한다. 더미↔실데이터 같은 **콘텐츠 텍스트 값 차이는 결함으로 보지 않는다.**
- 헤드리스 브라우저에 **디자인 폰트가 없으면** 폰트/시각 차이가 거짓양성일 수 있다(리포트에 표시됨).
- 로그인이 필요한 화면은 로그인 페이지가 캡처될 수 있다.
- 리포트는 임시 산출물 — 다시 돌리면 덮어쓴다.

## 문제 해결
| 증상 | 해결 |
|------|------|
| "접속 실패 / 서버 미기동" | 개발 서버를 먼저 띄운다(예: `npm run dev`) 후 재시도 |
| 캡처가 로그인/빈 화면 | 로그인 후의 URL을 주거나, QA할 화면 URL을 직접 지정 |
| 폰트가 다 다르게 보임 | 디자인 폰트 미설치 가능성 — 폰트 판정은 참고용으로 |
